# IAM — How-to guides

## Manage users

| Task | How |
|---|---|
| Invite a new user | Click the **Settings** gear icon (bottom-left of the sidebar), select **Users**, click **New Invite**, enter email and permission groups |
| Change a user's permission groups | Open the user's detail page, click **Edit**, update group assignments |
| Remove a user | Open the user's detail page, click **Remove**. Their resources remain but they lose access |
| Resend an invitation | Open the pending invitation from the Users page and click **Resend** |

## Manage permission groups

| Task | How |
|---|---|
| Create a custom group | Click the **Settings** gear icon (bottom-left of the sidebar), select **Permission Groups**, click **Create Group**, select the permissions to include |
| Assign a group to users | Open the group detail page and click **Add Members** |
| Clone an existing group | Use the group's action menu and select **Clone** |

## Configure authentication

| Task | How |
|---|---|
| Set up SSO | Click the **Settings** gear icon, select **Organization Settings**, scroll to **Custom Identity Provider**, and configure Azure/Google/GitHub |
| Generate an API key | Go to your profile settings and click **Create API Key** |

See [Concepts](/platform/iam/concepts) for background on permission groups and authentication methods.
