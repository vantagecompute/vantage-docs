# Managing Job Scripts

From a row's overflow menu or from the detail page header:

| Action | Effect |
|--------|--------|
| **Submit Script** | Opens the submit dialog so you can send this script to the cluster. |
| **Clone** | Creates an independent copy with a new name and description. |
| **Archive** | Hides from the default view. All submission references are preserved — the script still runs. |
| **Restore** | Returns an archived script to the active list. |
| **Delete** | Permanent. All files removed. References from past submissions are removed too. |

:::tip
**Archive vs. Delete.** Use archive when you might need the script again or when it has submissions you want to keep referencing. Delete is final — only do this for scripts you're certain are obsolete.
:::

## Filters

Filter by source (scripts you own or all scripts), by archived status, or search by name, ID, or owner.

## Bulk operations

Select multiple scripts using the checkboxes, then use the toolbar to archive, restore, or delete them all at once.

## Edit

Name, description, and files can be changed from the detail page. Click **Edit** to toggle the form.
