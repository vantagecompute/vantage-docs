# Sessions

Sessions are **JupyterLab, VS Code, or RStudio** instances pinned to your team's storage and a chosen compute profile. They're the fastest way onto a GPU on Vantage.

A session can also be a **[notebook](/platform/workbench/sessions/notebooks)** (a Jupyter-style web UI) or a **[remote desktop](/platform/workbench/sessions/remote-desktops)** (a full GUI environment) — both are session types and live in their own sub-pages.

## The sessions list

The list view shows everyone's sessions in your workspace. Each row gives you the name, image, compute profile, status, owner, and elapsed runtime.

Lifecycle actions — Open, Stop, Start, Edit, Delete — live on each row's overflow menu and on the detail page. See [Lifecycle](/platform/workbench/sessions/lifecycle) for what each one does.
