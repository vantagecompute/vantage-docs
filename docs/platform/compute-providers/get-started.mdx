# Quickstart

From zero to a connected compute provider in under five minutes.

1. **Open Cloud Accounts** — From the Vantage top nav, click **Admin**. Select **Cloud Accounts** from the left sidebar.
1. **Create an account** — Click <kbd>Create Cloud Account</kbd>, then pick a provider from the dropdown: AWS, Azure, GCP, Cudo Compute, LXD, or On-Premises.
1. **Name it** — Enter a unique account name and an optional description.
1. **Enter credentials** — Each provider needs different credentials. For AWS, use **Assisted Setup** to deploy a CloudFormation stack that creates the IAM role automatically — or paste an **IAM Role ARN** you've already created. For other providers, fill in the required fields — see [Azure](/platform/compute-providers/public-clouds/azure), [GCP](/platform/compute-providers/public-clouds/gcp), [Cudo Compute](/platform/compute-providers/public-clouds/cudo-compute), or [LXD and On-Premises](/platform/compute-providers/on-premises) for details.
1. **Submit** — Vantage validates the credentials and saves the account. It's now available when creating clusters.

:::tip
One cloud account can back multiple clusters. You don't need a separate account per cluster — but if you need compute in multiple AWS regions, you may want one account per region depending on how your IAM roles are scoped.
:::
